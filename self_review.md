# 자기 평가서

## 가장 잘했다고 생각하는 부분

이번 프로젝트에서 가장 잘했다고 생각하는 부분은 부모 클래스와 자식 클래스를 나누어
기능을 구성한 점입니다. `TextBase`에는 텍스트 저장과 토큰화 같은 공통 기능을 두고,
`TextAnalyzer`에서는 글자수, 문장수, 단어수 분석 기능을 구현하여 역할이 분명하게
나뉘도록 만들었습니다.

또한 `pytest`, `doctest`, `pycodestyle`을 실행하여 코드가 실제로 동작하는지와
스타일 규칙을 지키는지 확인했습니다. 단순히 코드를 작성하는 것에서 끝내지 않고,
테스트와 코드 품질 검사를 함께 진행한 점이 좋았다고 생각합니다.

## 아쉬운 부분

아쉬운 점은 처음부터 패키지 구조와 테스트 방식을 완전히 이해하고 시작하지는
못했다는 점입니다. 처음에는 테스트 파일에 `input()`을 사용했기 때문에 pytest가
자동으로 실행되기 어려웠고, 이후 테스트 함수와 `assert`를 사용하는 방식으로
수정해야 했습니다.

또한 문장 수를 계산하는 부분에서 처음에는 정규식 표현이 조금 복잡하게 작성되어
있었습니다. 나중에 `.` `!` `?`를 기준으로 나누는 더 단순한 방식으로 수정하면서,
코드는 기능뿐 아니라 읽기 쉬운 형태로 작성하는 것도 중요하다는 점을 알게 되었습니다.

## 다음에 개선하고 싶은 점

다음에는 문장 분석 기능을 조금 더 정확하게 개선하고 싶습니다. 현재는 마침표,
느낌표, 물음표를 기준으로 문장 수를 계산하지만, 줄바꿈이나 여러 문장 부호가 섞인
경우까지 더 자연스럽게 처리할 수 있도록 보완할 수 있을 것 같습니다.

또한 GitHub를 활용하여 작업 과정을 더 체계적으로 관리하고 싶습니다. 기능을 하나씩
추가할 때마다 커밋을 남기고, README와 테스트 결과도 함께 정리하면 프로젝트 진행
과정을 더 명확하게 보여줄 수 있을 것 같습니다.
