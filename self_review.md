# 자기 평가서

## 가장 잘했다고 생각하는 부분

이번 프로젝트에서 가장 잘했다고 생각하는 부분은 부모 클래스와 자식 클래스를 나누어
기능을 구성한 점입니다. `TextBase`에는 텍스트 저장과 토큰화 같은 공통 기능을 두고,
`TextAnalyzer`에서는 글자수, 문장수, 단어수 분석 기능을 구현하여 역할이 분명하게
나뉘도록 만들었습니다.

## 아쉬운 부분
아쉬운 점은 아직 AI의 도움에서 완전히 벗어날 수 없었다는 점입니다. Python 프로그래밍 수업에서 배운 내용과 제공된 자료를 최대한 활용하려고 노력했지만, 문제를 해결하는 과정에서 결국 AI에게 직간접적으로 도움을 받는 경우가 많았습니다. 처음에는 AI를 활용하면 더 빠르고 쉽게 결과물을 만들 수 있다는 점이 장점이라고 생각했지만, 프로젝트를 진행하면서 내가 스스로 문제를 분석하고 해결하는 능력이 아직 부족하다는 것을 깨닫게 되었습니다. 특히 오류가 발생했을 때 왜 문제가 생겼는지 정확히 이해하기보다 AI에게 해결 방법을 먼저 묻게 되는 경우가 있었고, 이 점에서 제 프로그래밍 기초와 문제 해결 역량을 더 키워야겠다고 느꼈습니다.
또한 제가 표현하고자 하는 기능을 코드로 자유롭게 구현하지 못했다는 점도 아쉬웠습니다. 원하는 결과는 머릿속에 있었지만, 그 기능을 만들기 위해 어떤 함수나 메서드를 사용해야 하는지 바로 떠올리지 못했고, 그로 인해 코드가 길어지거나 비효율적으로 작성되는 경우가 있었습니다. 더 간단하게 구현할 수 있는 부분도 있었지만, 아직 다양한 문법과 함수 활용에 익숙하지 않아 반복적인 코드나 불필요한 구조가 생겼습니다. 이번 경험을 통해 단순히 AI가 제시한 코드를 사용하는 것에 그치지 않고, 코드가 어떤 원리로 작동하는지 이해하고 직접 수정할 수 있는 능력이 중요하다는 것을 알게 되었습니다. 앞으로는 Python의 기본 문법뿐만 아니라 함수, 메서드, 코드 구조화 방법을 더 꾸준히 학습하여 스스로 문제를 해결할 수 있는 역량을 키우고 싶습니다.

## ai 사용
.isspace() 사용. 이 메서드는 문자열이 공백 문자인지 확인하는 메서드입니다.  ex)" ".isspace() 결과=TRUE "a".isspace() 결과=FALSE 제가 알고 있는 메서드중에선 공백인지 아닌지 계산하는 방법을 모르겠어 ai에게 이런 상황에서 공백인지 아닌지 확인하기위해서 어떤 걸 사용할 수 있는가를 물어보었습니다. self.text 안의 글자를 하나씩 보면서, 공백이 아닌 글자만 리스트에 넣기위해 사용하였습니다. 

.strip() 사용. 이 메서드는 문자열의 앞 뒤 공백을 제거하는 메서드 입니다. ex)" hello ".strip() 결과="hello" 혹시 문자 앞뒤에 공백이 있다면 그걸 안세고 싶지만 어떻게 해야하는지 모르겠어서 ai에게 조언을 구해 사용했습니다. 문장 리스트에서 실제 내용이 있는 문장만 새기 위하여 사용했습니다.
